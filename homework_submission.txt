Part 1: Written Proposal (4%)
========================================================================

Problem Statement This project utilizes the FronkonGames/steam-games-dataset to address the challenge of game discovery by answering the core user-facing question: "Given a game idea or preference written in plain English, what Steam games best match it, and what other games are similar?" This problem is especially interesting because Steam's massive and diverse library makes it difficult for users to find novel titles based on nuanced preferences using traditional, keyword-based search filters. By allowing users to express complex ideas naturally (e.g., "story-rich sci-fi game with controller support"), we can uncover hidden gems efficiently and provide a more intuitive, tailored browsing experience that goes beyond simple metadata matching.

Methodology To enable natural-language search and game discovery, we will apply several key algorithms from the course syllabus. First, we will utilize text embeddings (via a pre-trained model like sentence-transformers) to convert combined text metadata—such as game descriptions, tags, and genres—into dense vector representations, effectively capturing the semantic essence of each game. We will then implement a from-scratch cosine similarity and brute-force nearest neighbor retrieval algorithm to compute the distance between the user's embedded query and the game vectors, returning top-k matches. Furthermore, to support interactive exploration of the game space, we will apply dimensionality reduction using Principal Component Analysis (PCA) to project the high-dimensional embeddings into a 2D visual map, optionally combined with K-means clustering to automatically group visually similar games into segmented neighborhoods.

1. Repository Structure
Our codebase maintains a strict logical separation of concerns, broken down into the following operational modules:

demo1/
├── app.py                 - Main Streamlit frontend application integrating UI layout, search panels, and data visualizations.
├── data.py                - Data ingestion module for downloading the HuggingFace dataset and executing custom Regex cleaning.
├── embed.py               - Natural Language Processing module handling text concatenation and Sentence-Transformer vector embedding.
├── retrieval.py           - Core mathematical engine executing L2 normalization, pure vector dot products, and dealbreaker algebra.
├── clustering.py          - Unsupervised Machine Learning module implementing the K-Means algorithm for automated game grouping.
├── viz.py                 - Exploratory Data Analysis module running algorithmic PCA dimensionality reduction and rendering Plotly charts.
├── requirements.txt       - Standardized Python environment dependency list representing a fully reproducible execution state.
└── README.md              - Root documentation explaining project installation, execution commands, and fundamental capabilities.


2. Division of Labor
Based on our architectural separation of concerns, the workload is equitably distributed among the three team members:

* Student A (Data Engineering & Embeddings): Principally responsible for writing `data.py` and `embed.py`. This includes resolving the "dimension explosion" by extracting raw strings via Regex, utilizing Hash Table (`set`) deduplication for categorical variables, and managing the Sentence-Transformer integration to guarantee accurate high-dimensional representation.

* Student B (Core Algorithms & Vector Math): Assigned the `retrieval.py` engine. Key deliverables include bypassing high-level library abstractions by actively developing the L2 normalization logic, the Cosine-to-Dot-Product mathematical acceleration, the cross-genre vector midpoint search, and the advanced spatial repulsion algorithm for negative dealbreakers.

* Student C (Unsupervised ML & Data Visualization): Tasked with `clustering.py`, `viz.py`, and the ultimate `app.py` UI integration. Deliverables involve deploying the K-Means framework for genre-agnostic grouping, executing PCA mathematics to project high-dimensional data onto 2D interactive graphs, and interconnecting all reactive algorithms to the Streamlit layout.
